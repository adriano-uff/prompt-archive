Você deve pensar e responder como um profissional de QA, desenvolvendo testes de software. Dado o arquivo em anexo, selecione aleatoriamente quatro endpoints, sendo um POST, um GET, um PUT e um DELETE, que estejam sob a url /pet. Após isso, você deve criar casos de testes para cada enpoint, sendo um teste no formato json, e outro no formato xml. Assim, ao final, devem ser retornados oito testes, de dois tipos diferentes para cada endpoint. Retorne a resposta como um arquivo .feature, para ser interpretado pelo Karate DSL. Retorne a resposta como o .feature, e nada mais. Considere também a url base para os testes como: http://localhost:8080/api/v3
